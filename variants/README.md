## 字节跳动
### 出题偏好
- **整体偏好**：字节跳动算法面试 / 笔试以「LeetCode 高频原题 + 经典算法复合优化」为核心，既考察基础算法的代码落地与复杂度优化能力，也关注算法在核心业务场景的工程适配（如视频分发、实时计算、用户行为序列处理）。笔试时长通常 2 小时，题量按岗位差异化设置：算法岗为「选择题 + 4 道编程题」，后端岗为「选择题 + 3 道编程题」，客户端 / 测试岗为「选择题 + 4 道编程题」，前端 / 数据岗为「选择题 + 2-3 道编程题」。整体难度呈递进式分布，前 1-2 题以 Easy-Medium 基础题为主（如数组双指针、简单递归），最后 1 题达 Medium-Hard 难度（多为算法复合或复杂度优化题），编程题输入规模侧重千万级（1e7）流数据处理（如用户行为序列），对 O (n) 线性时间要求极严，需规避嵌套循环（适配实时推荐场景）。技术一面聚焦「算法正确性与代码落地」，要求手撕基础算法（如快排、滑动窗口）并处理边界条件，二面侧重「算法复合与优化能力」（如多算法融合、空间复杂度压缩），算法岗二面及以后会加入 AI/ML 相关考察，三面核心是「系统扩展与业务适配」（如算法在分布式场景的落地、性能指标量化）。

  - **高频主题占比：**
  数组 / 字符串（28%）、动态规划（DP）（18%）、树结构（15%）、贪心（7%）、图论（10%）、其他（链表 / 堆 / 排序 / 并查集 / 数学位运算）（22%）。其中数组 / 字符串是绝对基础高频，以双指针、滑动窗口、前缀和为核心考点；堆（优先队列）考察频率高于纯搜索，多应用于 TopK、数据流中位数等场景；排序算法常以「手撕快排 / 归并排序」形式直接考察，是基础得分点。
  - **趋势：**
  大模型相关题目集中在算法岗（尤其 NLP / 推荐方向），形式多为「大模型辅助传统算法」（如用 RAG 生成用户特征辅助推荐排序）或工程化落地（如大模型微调优化、LangChain 框架应用），非算法岗极少涉及纯大模型原理题；算法工程化考察强化，侧重「复杂约束下的优化」（如 O(1) 空间适配、分布式场景算法调整），但经典基础（如 DP 状态转移、贪心正确性证明）仍是核心评判标准。

### 岗位偏好
- 后端（e.g., 内容分发系统、飞书协作、抖音云）：数组 / 字符串（28%）+ 图 / DP（30%），高频题包括「区间 DP 多约束优化」「子数组和的滑动窗口」「图的连通分量计算」，典型高频题如 K 个一组翻转链表、无重复字符的最长子串、LRU 缓存机制，侧重「高并发与大规模数据适配」（如分布式日志的负载均衡、CDN 路径规划的图算法优化），笔试常出现「算法复合题」（如并查集 + 排序解决区间合并问题）。
- 算法岗（推荐 / 广告 / 搜索 / NLP）：DP / 贪心（25%）+ ML / 大模型融合（15%），高频题涵盖「用户行为序列的 DP 建模」「推荐多样性的贪心优化」「Transformer 核心模块手撕」（如 multi-head self-attention、beam search 实现），侧重「算法与业务指标的关联」（如用贪心提升推荐点击转化率），面试会深入考察大模型实践（如 RAG 优化、Prompt 工程、模型微调方法）。
- 客户端（e.g., 抖音 App、头条客户端）：数组 / 字符串 / 链表（34%），高频题包括「数组双指针匹配」「链表条件反转」「字符串匹配与处理」，侧重「端侧高效实现」（如 O(1) 空间的本地缓存检索、滑动界面的性能优化），笔试多基础算法原题或简单变式（如合并两个有序数组、翻转二叉树），面试聚焦代码简洁性与执行效率。
- 数据开发 / 数仓：图 / DP / 并行计算（41%），高频题包括「大规模图的连通分量计算」「分布式环境下的前缀和统计」「亿级数据的去重与聚合」，适配 Flink/Spark 框架，侧重「TB 级数据处理的鲁棒性」（如离线数据的 DP 批量计算、图算法的并行化实现），笔试常考察「算法与大数据框架的结合」（如用 MapReduce 实现图遍历）。

## 阿里巴巴
### 出题偏好
- **整体偏好**：
阿里巴巴算法面试/笔试以「LeetCode 企业高频题 + 电商/搜索业务场景适配」为核心，既考察经典数据结构与算法的实现与优化（如时间/空间复杂度分析），更注重算法在淘宝/天猫推荐、搜索排序、实时风控等核心业务中的应用（如商品排序、用户画像预测）。笔试时长通常 120 分钟，题量因岗位差异明显：算法岗为「15 道选择题 + 3-4 道编程题」，后端岗为「15 道选择题 + 3 道编程题」，客户端/数据岗为「15 道选择题 + 2-3 道编程题」，难度渐进（前 1-2 题 Easy-Medium，最后 1 题 Medium-Hard），且编程题输入规模常达 1e5 级别，需重点规避暴力解。技术一面强调「代码正确性与边界处理」，要求手撕高频题（如两数之和、二叉树中序遍历）并讨论海量数据下的优化（如 O(n log n) 排序在亿级商品中的适用性），二面聚焦「业务融合与扩展」（如算法如何提升搜索召回率），三面引入「系统设计与量化评估」（如在分布式环境下的图算法部署对延迟的影响）。
  - **高频主题占比**：数组/字符串（32%）、动态规划（DP）（28%）、树/图（18%）、哈希表/贪心（15%）、其他（链表/堆/并查集/二分）（7%）。其中数组/字符串以双指针、滑动窗口为核心，DP 侧重基础经典题（如最大子序和、爬楼梯），树题聚焦遍历与 BST 性质验证，均为面试必考点。
  - **趋势**：大模型与搜索推荐融合题占比上升（如基于 BERT 的字符串匹配优化、用 GNN 提升商品图谱覆盖），多模态大模型相关考察在算法岗增多（如 CLIP 原理、InfoNCE Loss 实现），AI 工程化考察增多（如模型推理的并行适配、联邦学习下的隐私图算法），但经典基础（如 DP 状态压缩、贪心证明、树遍历变体）的业务验证仍是核心底线。
- **岗位偏好：**
  - 后端（e.g., 淘宝、天猫、阿里云）：数组/DP/哈希表（45%），变体多为「多约束条件」（如带权重的背包变体），高频题包括两数之和、最长无重复字符子串、验证二叉搜索树，业务场景如订单排序、库存调度、分布式缓存优化、分布式协议适配（如 Paxos 相关算法落地）。
  - 算法岗（推荐/广告/搜索/多模态）：图/贪心/DP（40%）+ 大模型基础（15%），侧重「大规模数据适配」（如亿级用户的最短路径变体），融合 ML 特征工程（如基于用户行为的序列 DP），面试会考察多模态模型原理（如 BLIP、LLaVA 结构）、Attention 实现、RLHF 思路等。
  - 客户端（e.g., 支付宝、钉钉）：字符串/链表/树（35%），偏「内存优化型变体」（如 O(1) 空间的链表操作），高频题涵盖反转链表、合并两个有序链表、二叉树遍历，业务场景如本地数据同步、界面渲染优化、端侧缓存检索。
  - 大数据/数仓：分治/外排序/哈希（30%）+ 分布式算法（20%），核心考察「海量数据处理」（如超大数据组的逆序对变体、分布式环境下的最大子序和），适配 Hadoop/Spark 框架，业务场景包括数据倾斜优化、MapReduce 二次排序、两表 Join 实现、HBase 数据读写算法适配。

## 华为
### 出题偏好
- **整体偏好**：
华为算法面试/笔试以「LeetCode 高频题 + 通信/云/AI 系统场景融合」为核心，既考察经典算法的工程实现与硬件优化（如时间/空间复杂度在嵌入式环境下的适配），更注重算法在5G/芯片设计、云存储、AI边缘计算等核心场景的落地（如信号排序、网络拓扑预测）。笔试时长通常 120 分钟，题量因岗位差异明显：算法岗为「选择题 + 4 道编程题」，后端岗为「选择题 + 3 道编程题」，客户端/无线岗为「选择题 + 2-3 道编程题」，难度递进（前 1-2 题 Easy-Medium，最后 1 题 Hard），且编程题输入规模**聚焦千万级（1e7）信号帧/网络节点数据**（如5G基站频谱监测），对**嵌入式场景的内存限制极严**（常要求 O(1) 或 O(log n) 空间复杂度，适配低功耗设备）。技术一面强调「代码正确性与边界鲁棒性」，要求手撕高频题（如数组旋转、Dijkstra 最短路径）并分析低功耗设备下的优化（如 O(1) 空间在亿级信号数据中的应用），二面聚焦「系统融合与扩展」（如算法如何提升5G延迟），三面引入「硬件/分布式设计与量化评估」（如算法在鲲鹏芯片上的并行执行对吞吐量的影响）。
  - **高频主题占比**：数组/字符串（25%）、动态规划（DP）（22%）、树/图（22%）、贪心（15%）、其他（搜索/堆/ML 基础/链表）（16%）。其中数组/字符串侧重「信号序列处理」（如频谱数据滑动窗口分析），树/图聚焦「网络拓扑与路径容错」（如基站通信链路设计），DP 多围绕「资源实时调度」（如芯片算力分配），均为笔面必考点。
  - **趋势**：5G/AI 与芯片融合题占比激增（如基于 GNN 的网络图优化、用 Transformer 提升信号序列预测），硬件工程化考察强化（如 ARM/鲲鹏适配的并发算法、隐私计算下的 DP），边缘计算场景的「低功耗算法设计」占比显著（如端侧模型推理的量化优化），但经典基础（如 DP 状态压缩、图最短路径证明、树遍历变体）的系统验证仍是底线。相比京东，华为更偏通信图算法（占比高 4%），场景如基站调度中 DP 应用更注重实时性。
- **岗位偏好：**
  - 后端（e.g., 华为云服务、存储系统、分布式调度）：DP/贪心/图（38%），变体多为「高可靠优化问题」（如容错路径规划变体），高频题包括合并区间、最小生成树（Kruskal 变体）、子数组和的流量滑动窗口，业务场景如云存储 CDN 调度、分布式数据库负载均衡、实时故障异常检测、基站通信链路自愈算法。
  - 算法岗（推荐/搜索/AI）：DP/贪心/ML 融合（40%），侧重「信号/数据驱动的模型设计」（如基于传输轨迹的字符串 DP 预测变体），融合特征工程（如网络延迟/频谱特征融入贪心状态），常考察「算法与 KPI 的量化关联」（如如何用模型提升 AI 推理准确率或降低云延迟），面试会涉及 MindSpore 框架适配、5G 信号调制解调算法、图神经网络在网络拓扑预测中的应用。
  - 客户端（e.g., 鸿蒙 App、终端设备）：数组/字符串/链表（32%），偏「低功耗端侧实现」（如 O(1) 空间的本地信号解析），高频题涵盖反转链表、数组双指针的滑动匹配、字符串压缩（哈夫曼编码变体），业务场景如 App 5G 预加载的增量同步、用户隐私标签的快速检索、滑动界面功耗调优、鸿蒙终端的本地数据加密算法。
  - 数据开发/数仓：图/DP/并行计算（39%），重点评估「海量数据清洗与融合」（如TB级信号日志的图神经网络变体），兼容 Spark/MindSpore 平台，高频场景包括 Spark 实现基站流量实时统计、MindSpore 分布式训练的数据集分片、图计算引擎处理通信拓扑图，实际应用包括多源数据湖的异构聚合、边缘AI轨迹的在线推理、模型训练集的异常过滤与实验迭代验证。
